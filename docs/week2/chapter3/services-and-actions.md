# Services and Actions

Request-response and long-running task patterns in ROS 2.

## Overview

Services and actions provide synchronous and asynchronous communication patterns beyond publish-subscribe.

## Services

- **Pattern**: Request-response
- **Use Case**: Quick, one-time queries
- **Blocking**: Client waits for response

## Actions

- **Pattern**: Goal-feedback-result
- **Use Case**: Long-running tasks
- **Features**: Preemption, progress feedback
